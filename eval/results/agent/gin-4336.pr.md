# fix(recovery): suppress and re-panic on http.ErrAbortHandler

The recovery middleware was catching `http.ErrAbortHandler`, which is a sentinel panic intended to abort a handler and be handled by the HTTP server itself. Recovering it caused unwanted logging and stack traces, and prevented the server from aborting the connection as expected. This fix detects `http.ErrAbortHandler` in the deferred recovery and re-panics it so the server can handle it normally.
