# Add VarWithKeyCtx method to include field keys in map validation errors

The `ValidateMapCtx` function was calling `VarCtx` which returned validation errors without field key information, making error messages unhelpful (e.g., "Key: '' Error:Field validation for '' failed..."). This fix introduces a new `VarWithKeyCtx` method that accepts a key parameter and properly populates the namespace fields (`vd.ns` and `vd.actualNs`) before validation. The `ValidateMapCtx` function now calls `VarWithKeyCtx` instead, ensuring error messages include the correct field keys while maintaining backward compatibility with the existing `VarCtx` method.
